# Описание админки

Команда пересборки, которую можно просто в терминале запускать или добавить в конфигурацию PHPStorm:

```bash
docker compose exec node sh -c 'cd /home/node/app/backend/views/assets/media && npm run build'
```

В PHPStorm добавлять в раздел 'Shell script' -> 'Script text'. Рабочая директория: корень приложения.

## Жесты

TS-модуль `src/js/swipe-navigation.ts` подключён в `index.ts` (импорт + вызов `initSwipeNavigation()` 
внутри `DOMContentLoaded`).

Жесты (один палец, тач-устройства):

- свайп вправо от левого края → открыть левый сайдбар `#sidebarMenu`;
- свайп влево от правого края → открыть правый сайдбар `#rightSidebarMenu`;
- свайп в обратную сторону по открытому сайдбару → закрыть его.

Ключевые решения:

- Управление через штатный `Bootstrap` Offcanvas API (`getOrCreateInstance().show()/.hide()`) — значит бэкдроп,
  блокировка скролла `body`, тап-по-фону и доступность работают как при клике по кнопке. Ничего в разметке менять не
  пришлось.
- Левый сайдбар — responsive `offcanvas-md`: выше `md` (768px) он статичная колонка, а не offcanvas. Поэтому
  свайп-открытие для него включается только при ширине `< 768`, чтобы не создавать «фантомный» бэкдроп на десктопе.
  Правый (`offcanvas-end`) — offcanvas всегда, работает на любой ширине.
- Защита от ложных срабатываний: жест засчитывается только если он выраженно горизонтальный
  (`|dx| > |dy|·1.5` и `≥ 60px`), старт открытия — из 32px-зоны у края.
- Свайп внутри горизонтально прокручиваемого контейнера (мобильная карточная таблица GridView) игнорируется — чтобы не
  конфликтовать с прокруткой таблиц.
- Слушатели `passive: true`, `preventDefault` не вызывается — вертикальный скролл страницы не блокируется.

Все листенеры навешены один раз на document, так что работает на любой странице админки.

Обратите внимание: на десктопе без тача жесты неактивны в принципе
(это `touchstart/touchend`) — это соответствует «в телефоне».
